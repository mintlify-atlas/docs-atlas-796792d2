Get a local Hiero consensus node compiling, passing quality checks, and serving gRPC traffic in a few commands.

## Prerequisites

- **JDK 25** — Install the [Eclipse Adoptium](https://adoptium.net/) build of JDK 25.0.2. The Gradle build enforces this version and will fail with a clear error if your `java` command resolves to a different JDK. Set `JAVA_HOME` or update your `PATH` accordingly.
- **Git** — Any recent version.

<Note>
  You do not need to install Gradle separately. The repository includes a `gradlew` wrapper script that downloads the correct version automatically and scopes it to this project.
</Note>

## Build and run

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/hiero-ledger/hiero-consensus-node.git
    cd hiero-consensus-node
    ```
  </Step>

  <Step title="Compile and assemble all JARs">
    ```bash
    ./gradlew assemble
    ```

    This compiles every module and produces all JAR files. On a clean checkout with a warm Gradle daemon this typically takes a few minutes.
  </Step>

  <Step title="Run quality checks">
    ```bash
    ./gradlew qualityGate
    ```

    Runs all static analysis checks and auto-fixes formatting where possible. All checks must pass before a pull request can be merged.
  </Step>

  <Step title="Start a local node">
    ```bash
    ./gradlew :app:modrun
    ```

    Starts a single-node services instance. The node exposes a gRPC endpoint on **port 50211** — use this port when pointing any Hedera SDK or client at your local node.

    <Tip>
      You can also invoke any Gradle task from the **Gradle** tool window in IntelliJ IDEA instead of the command line. See the [IntelliJ IDEA quickstart](https://github.com/hiero-ledger/hiero-consensus-node/blob/main/docs/intellij-quickstart.md) for IDE setup instructions including recommended plugins and run configurations.
    </Tip>
  </Step>

  <Step title="Run a test client">
    With the node running, open a second terminal and execute:

    ```bash
    ./gradlew :test-clients:runTestClient \
      -PtestClient=com.hedera.services.bdd.suites.crypto.HelloWorldSpec
    ```

    `HelloWorldSpec` creates a simple crypto account and verifies the response. A successful run confirms that gRPC ingestion, consensus, and state handling are all working end-to-end.
  </Step>
</Steps>

## Running individual test suites

The project uses several test types, each in its own Gradle source set:

| Type | Source folder | Gradle task |
|---|---|---|
| Unit tests | `src/test` | `./gradlew :<module>:test` |
| Integration tests | `src/testIntegration` | `./gradlew :<module>:itest` |
| Hammer (fuzz) tests | `src/hammer` | `./gradlew :<module>:hammer` |
| Micro-benchmarks | `src/jmh` | `./gradlew :<module>:jmh` |

Unit and integration tests run on every pull request and must pass before merging. Hammer tests and benchmarks run nightly.
