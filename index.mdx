Hiero Consensus Node is the reference implementation of a consensus node for Hiero-based distributed ledger networks. Each node participates in the [hashgraph](https://www.swirlds.com/downloads/SWIRLDS-TR-2016-01.pdf) consensus algorithm to order transactions fairly and securely, then executes those transactions to mutate shared network state identically across all nodes.

## What is a consensus node?

A consensus node gossips with every other node in the network to reach agreement on which transactions should execute and in what order. Once consensus is reached, each node executes the transactions in that order, updating its local copy of the state in lockstep with all other nodes.

Clients interact with nodes through the **Hedera API (HAPI)** — a gRPC interface defined by Protocol Buffers. HAPI exposes two kinds of operations:

- **Transactions** — requests to mutate network state (e.g. transfer tokens, create a topic)
- **Queries** — requests to read current network state (e.g. get account balance, fetch file contents)

## Getting started

<Steps>
  <Step title="Install prerequisites">
    Install an [Eclipse Adoptium](https://adoptium.net/) build of **JDK 25**. No other tooling is required — the repository includes a self-contained Gradle wrapper.
  </Step>
  <Step title="Build the project">
    Clone the repository and run:
    ```bash
    ./gradlew assemble
    ```
  </Step>
  <Step title="Run a local node">
    Start a single-node services instance:
    ```bash
    ./gradlew :app:modrun
    ```
    The node exposes a gRPC server on port `50211` by default.
  </Step>
  <Step title="Submit your first transaction">
    Use any [Hedera SDK](https://docs.hedera.com/hedera/sdks-and-apis/sdks) or the `test-clients` module to submit a `CryptoTransfer` or query an account balance.
  </Step>
</Steps>
